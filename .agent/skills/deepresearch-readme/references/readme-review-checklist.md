# README Review Checklist

## 1. First View

- [ ] H1がプロジェクト名になっている
- [ ] 1〜2文で「何ができるか」が書かれている
- [ ] 読者の価値（Why）が先に書かれている
- [ ] 主要導線（Quickstart / Docs / Issues）がある

## 2. Quickstart

- [ ] `Requirements` が明記されている
- [ ] `Install` が明記されている
- [ ] `Run` または最小実行例がある
- [ ] 手順が実行可能で、不要な分岐が少ない

## 3. Core Sections

- [ ] `Usage` がある
- [ ] `Documentation` への導線がある
- [ ] `Getting help`（Issue/Discussion等）がある
- [ ] `Contributing` への導線がある
- [ ] `License` が明記されている

## 4. Structure

- [ ] READMEは導入に必要な情報へ絞られている
- [ ] 長文の詳細は `docs/` へ分離されている
- [ ] リンクは相対リンクを優先している
- [ ] 見出し構造が整理されている

## 5. Quality

- [ ] 古いコマンドや無効なリンクがない
- [ ] バージョン前提が明示されている
- [ ] 用語が一貫している
- [ ] 画像やバッジが過剰でない

## Severity Guide

### Critical

- 最小実行手順が成立しない
- READMEだけで開始判断ができない

### Warning

- 情報順が悪く、価値や導入が見つけにくい
- docsへの導線が弱い

### Info

- 表現や具体例の改善余地がある
